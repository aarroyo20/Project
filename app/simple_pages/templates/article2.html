AAA

A pattern for writing good and useful tests

Arrange

This step should set up the test case. This is where objects are set up to be tested. In my tests, this is where the tuple_list = (1,2) comes from.

Act

This step should cover the main thing to be tested and is where you act upon the system under test.. This is where the calling a function or method happens. This is where you pass the dependencies and capture the output value if any.

Assert

This step should verify the quality of the response that is elicited in the previous Act step. This step may be as simple as just verifying and checking numeric or string values. There are cases where it may require checking multiple facets of a system. This step is the deciding determination if the test is passing or failing.


Unit Testing

Unit tests are essential to software development. A unit test can check multiple behaviors of a system, but a unit test can fall under two categories: state-based and interaction-based. State-based unit testing verifies that the system produces correct results; whereas, interaction-based unit testing makes sure that the system calls the correct methods.
