CI/CD

CI/CD stands for continuous integration and continuous delivery/continuous deployment. This is a modern software development practice in which incremental code changes are made frequently and reliably. Therefore, automated build-and-test steps triggered by CI ensure that code changes being merged into the repository are reliable. The code is then delivered quickly and seamlessly as a part of the CD process. This pipeline refers to the automation that enables incremental code changes from developersâ€™ desktops to be delivered quickly and reliably to production.





CI/CD is important because it allows for a developer/organizations to delivery software quickly and efficiently. It is an effective process for getting products to market quicker than previous ways and allows continuous delivery of code into production. It ensures an ongoing flow of new features and fixes through a fast an efficient method.
My project has two workflows for continuous integration and delivery. My project has the production workflow and the development workflow which both have the goal to deploy the app. The production workflow will eventually create a docker image that is deployed in heroku as an app.
